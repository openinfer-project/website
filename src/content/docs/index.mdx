## Why openinfer

<CardGrid stagger>
	<Card title="Rust + CUDA, nothing else" icon="seti:rust">
		The entire stack — weight loading, paged KV cache, schedulers, kernels —
		is built from the ground up in Rust and CUDA, with Triton AOT and
		FlashInfer kernels compiled at build time. Memory-safe orchestration,
		hand-written kernels, and not a byte of Python in the serving path.
	</Card>
	<Card title="OpenAI-compatible API" icon="rocket">
		Serves a `/v1/completions` endpoint with streaming. Point any OpenAI SDK
		or `curl` at it and start generating — no client changes required.
	</Card>
	<Card title="One engine per model" icon="puzzle">
		No universal model abstraction. Each model line owns its scheduler,
		kernel plan, and execution path — full attention, hybrid linear
		attention, MLA, and MoE with expert parallelism.
	</Card>
	<Card title="CUDA Graph decode" icon="setting">
		The decode path is captured as a CUDA graph with pre-allocated buffers,
		keeping per-token overhead low and decode latency flat across context
		lengths.
	</Card>
	<Card title="Blogs" icon="rocket">
		Read the [latest posts](/blog/) for release notes, benchmark writeups,
		runtime design notes, and model bring-up stories.
	</Card>
</CardGrid>

## Talk to it like OpenAI

```bash
curl -s http://localhost:8000/v1/completions \
  -H "Content-Type: application/json" \
  -d '{"prompt": "The capital of France is", "max_tokens": 32}'
```

Set the base URL to `http://localhost:8000/v1` and any OpenAI SDK works
unchanged. Add `"stream": true` for token-by-token streaming.
